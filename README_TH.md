# OpenCode (รุ่นปรับปรุง)

> **⚠️ คำประกาศ**: โปรเจกต์นี้แยกมาจาก [opencode](https://github.com/sst/opencode) และดูแลโดยนักพัฒนาอิสระ ไม่มีความเกี่ยวข้องกับ **ทีม OpenCode อย่างเป็นทางการ** ไม่มีสายสัมพันธ์ใด ๆ โปรเจกต์ต้นฉบับเผยแพร่โดยทีม opencode ภายใต้ใบอนุญาต MIT สาขาใหม่นี้ยังคงรักษาใบอนุญาต MIT ต้นน้ำไว้พร้อมเพิ่มโมดูลที่พัฒนาเองหลายตัว (ดูรายละเอียดที่ [NOTICE](./NOTICE))

## บทนำ

โปรเจกต์นี้คือ**รุ่นปรับปรุงและเพิ่มเติม**จาก OpenCode เวอร์ชันทางการ มีเป้าหมายคือ:

- 🔧 **แก้ไขปัญหาเกี่ยวกับภาษาจีน**: แก้ไขความเข้ากันได้ของการแบ่งคำภาษาจีน การประมวลผลตัวอักษร CJK เครื่องหมายวรรคตอนแบบ full-width เส้นทางภาษาจีน และสถานการณ์การป้อนข้อมูลด้วย IME ภาษาจีนใน upstream (ดูรายละเอียดที่ [รายการแก้ไขคุณสมบัติภาษาจีน](./docs/localization/zh-hans-fixes.md))
- 🧩 **มอบเครื่องมือ DAG Workflow ระดับโปรดักชัน**: พัฒนาเอง [Harness-DAG-Workflow](./docs/harness-dag.md) อนุญาตให้ LLM agent จัดการและขับเคลื่อนงานแบบขนานหลายโหนดในเซสชันเดียว
- 🎯 **รักษาความเข้ากันได้กับต้นน้ำ**: โค้ดทั้งหมดภายใต้ใบอนุญาต MIT ของต้นน้ำยังคงเหมือนเดิม ไม่ทำลายการสร้างเดิม ไม่ทำให้ API ต้นน้ำเสียหาย

## การติดตั้ง

```bash
# ติดตั้งโดยตรง (YOLO)
curl -fsSL https://opencode.ai/install | bash

# ตัวจัดการแพ็กเกจ
npm i -g opencode-ai@latest        # หรือใช้ bun/pnpm/yarn
scoop install opencode             # Windows
choco install opencode             # Windows
brew install anomalyco/tap/opencode # macOS และ Linux (แนะนำ อัปเดตอยู่เสมอ)
brew install opencode              # macOS และ Linux (สูตร brew ทางการ ความถี่อัปเดตต่ำกว่า)
sudo pacman -S opencode            # Arch Linux (Stable)
paru -S opencode-bin               # Arch Linux (ล่าสุดจาก AUR)
mise use -g opencode               # ทุกระบบ
nix run nixpkgs#opencode           # หรือใช้ github:anomalyco/opencode สำหรับสาขาล่าสุด
```

> [!TIP]
> ก่อนติดตั้ง กรุณาลบเวอร์ชันเก่าก่อน 0.1.x ออกก่อน

## คุณสมบัติเฉพาะของสาขานี้

สาขานี้สร้างขึ้นจาก opencode ต้นน้ำ โดย**เพิ่มเติม**หรือ**ปรับปรุงอย่างมีนัยสำคัญ**ซึ่งความสามารถต่อไปนี้ (รายละเอียดในแต่ละส่วน):

| คุณสมบัติ | คำอธิบายโดยย่อ | สัญญาอนุญาต |
|------|------|------|
| 🧩 ระบบประสานงาน DAG HARNESS | ให้ LLM agent จัดเรียงเวิร์กโฟลว์แบบหลายโหนดคู่ขนานในเซสชันเดียว | AGPL-3.0 |
| 🪝 การใช้งาน HOOKS API แบบซูเปอร์เซ็ต | ระบบ Hooks ที่สมบูรณ์ด้วยเหตุการณ์รันไทม์ 22 ชนิด × ประเภทการดำเนินการ 5 แบบ | MIT + ส่วนขยายของสาขานี้ |
| 🛡️ พื้นที่แยกส่วนสำหรับการเขียนโค้ดแบบเบา | สภาพแวดล้อมการดำเนินการแยกส่วนแบบคู่: Sandbox + Worktree | MIT + ส่วนขยายของสาขานี้ |
| 🔧 แก้ไขข้อบกพร่องด้านภาษาจีน | การตัดคำ CJK / เครื่องหมายวรรคตอนแบบเต็มความกว้าง / ความเข้ากันได้ของ IME / พาธภาษาจีน | MIT |
| 🔬 แก้ไขข้อบกพร่องเล็กน้อยอื่นๆ | การคัดลอกวาง, ความกว้างอักขระภาษาตะวันออก, การตัดข้อความภาษาจีน ฯลฯ | MIT |

### 🧩 ระบบประสานงาน DAG HARNESS (โมดูลที่พัฒนาขึ้นเอง · AGPL-3.0)

แต่เดิมคือ Harness-DAG-Workflow เป็น**เอนจินเวิร์กโฟลว์แบบกราฟไม่มีวัฏจักร (DAG)** ระดับโปรดักชัน ที่ช่วยให้ LLM agent สามารถจัดเรียงงานคู่ขนานที่ซับซ้อนภายในเซสชันเดียว ความสามารถหลัก:

- **การจัดตารางอัตโนมัติ**: สร้าง agent ย่อยโดยอัตโนมัติตามความสัมพันธ์ของโหนดและดำเนินการแบบคู่ขนาน
- **การวางแผนใหม่แบบพลวัต**: สามารถวางแผนเวิร์กโฟลว์ใหม่ขณะดำเนินการได้ (เพิ่ม/ลบ/แก้ไขโหนด, ปรับขีดจำกัดการทำงานพร้อมกัน)
- **การปฏิบัติตามกฎอย่างเคร่งครัด**: สเตตแมชชีนไม่สามารถเลี่ยงได้, สถานะสุดท้ายย้อนกลับไม่ได้, เหตุการณ์ต้องถูกแพร่สัญญาณ, ให้ความสำคัญกับการบันทึกข้อมูลถาวร
- **การรวมคำสั่ง Slash**: `/dag-ctl` ควบคุมการทำงาน, `/dag-worker` กำหนดค่าเวิร์กโฟลว์
- **การตรวจสอบย้อนหลังถาวร**: สกีมา SQLite 6 ตาราง ทุกการเปลี่ยนแปลงสถานะสามารถตรวจสอบย้อนหลังได้

สถาปัตยกรรมฉบับสมบูรณ์ดูได้ที่ [เอกสาร Harness-DAG-Workflow](./docs/harness-dag.md), แนวทางการพัฒนาดูได้ที่ [AGENTS.md](./packages/opencode/src/dag/AGENTS.md)

> **สัญญาอนุญาต**: โมดูลนี้ ([`packages/opencode/src/dag/`](./packages/opencode/src/dag/), [`packages/opencode/src/tool/dag-worker.ts`](./packages/opencode/src/tool/dag-worker.ts), [`packages/opencode/src/tool/node_complete.ts`](./packages/opencode/src/tool/node_complete.ts) และแม่แบบกับเอกสารที่เกี่ยวข้อง) เผยแพร่ภายใต้สัญญาอนุญาต **GNU AGPL v3** — การใช้โมดูลนี้ต้องเปิดเผยซอร์สโค้ดของการแก้ไขทั้งหมด ดูรายละเอียดใน [NOTICE](./NOTICE) และ [packages/opencode/src/dag/LICENSE](./packages/opencode/src/dag/LICENSE)

### 🪝 การใช้งาน HOOKS API แบบซูเปอร์เซ็ต

สาขานี้คงรักษาและปรับปรุงระบบ Hooks API จากต้นน้ำอย่างสมบูรณ์:

- **เหตุการณ์ทริกเกอร์ขณะรัน 22 ชนิด**: `PreToolUse` / `PostToolUse` / `FileChanged` / `UserPromptSubmit` / `Stop` / `SessionStart` / `SessionEnd` / `PermissionRequest` / `PermissionDenied` / `SubagentStart` / `SubagentStop` / `TaskCreated` / `TaskCompleted` / `TeammateIdle` / `PreCompact` / `PostCompact` / `WorktreeCreate` / `WorktreeRemove` / `ConfigChange` / `InstructionsLoaded` / `StopFailure` / `PostToolUseFailure`
- **ประเภทการดำเนินการ 5 ชนิด**: `command` (เชลล์) / `mcp` (เครื่องมือ MCP) / `http` (REST) / `prompt` (LLM รอบเดียว) / `agent` (LLM หลายรอบ)
- **โปรโตคอลการสื่อสารแบบซอง JSON ผ่าน stdin/stdout**: เอกสารโปรโตคอลฉบับสมบูรณ์ดูได้ที่ [`hooks-reference.md`](./packages/opencode/src/session/prompt/hooks-reference.md)
- **การปรับปรุงของสาขานี้**: การรวมเข้ากับบัสเหตุการณ์เวิร์กโฟลว์ DAG (เหตุการณ์ `workflow.*` / `node.*`) + การสมัครรับผ่าน TUI + การส่งต่อ HTTP API

### 🛡️ พื้นที่แยกส่วนสำหรับการเขียนโค้ดแบบเบา

สาขานี้มอบสภาพแวดล้อมการดำเนินการแยกส่วนแบบคู่ เพื่อให้ agent/ผู้ใช้ทดลองรันโค้ดในแซนด์บ็อกซ์ที่ปลอดภัยโดยไม่ปนเปื้อนที่เก็บจริง:

| ระดับการแยกส่วน | กลไก | การใช้งาน |
|---------|------|------|
| **Sandbox** (เบา) | ไดเรกทอรีชั่วคราว + การวินิจฉัย LSP + ชุดเครื่องมือหลายภาษา (Python/Node/TS/Go/Rust/C/C++) | การทดลองรันโค้ดไฟล์เดียวหรือการทดลองเล็ก |
| **Worktree** (หนัก) | `git worktree` สาขาอิสระ + มุมมองระบบไฟล์แยกต่างหาก | การแก้ไขคู่ขนานของหลาย agent, การรีแฟกเตอร์ขนาดใหญ่ |

- 📦 **เครื่องมือ Sandbox**: `packages/opencode/src/tool/sandbox.ts`, แต่ละแซนด์บ็อกซ์มีแคชดีเพนเดนซีอิสระ (venv / node_modules), รองรับโหมดใช้ครั้งเดียวทิ้ง `ephemeral` และงานระยะยาวแบบอะซิงโครนัส `background`
- 🌳 **ตัวจัดการ DAG Worktree**: ในเวิร์กโฟลว์ DAG แต่ละโหนดคู่ขนานสามารถถูกจัดสรรไปยังสาขา worktree อิสระโดยอัตโนมัติ เมื่อโหนดเสร็จสมบูรณ์จะรวมเข้าสายหลักผ่าน `git merge`

### 🔧 แก้ไขข้อบกพร่องด้านภาษาจีน (ปัญหาต้นน้ำที่ได้รับการแก้ไข)

ทำการแก้ไขข้อบกพร่องและปรับปรุงปัญหาความเข้ากันได้/ประสบการณ์การใช้งานที่พบในกรณีใช้งานภาษาจีนสำหรับเวอร์ชันต้นน้ำ ครอบคลุม:

- **การตัดคำภาษาจีนและการนับโทเค็น**: การจัดการอักขระ CJK ที่ผิดปกติในบางโทเค็นไนเซอร์
- **ความเข้ากันได้ของเครื่องหมายวรรคตอนแบบเต็มความกว้าง**: การยอมรับข้อผิดพลาดของโคลอน อัญประกาศ วงเล็บแบบเต็มความกว้างในการแยกวิเคราะห์คอนฟิก
- **การจัดการพาธภาษาจีน**: การส่งผ่านพาธไฟล์ที่มีช่องว่างและอักขระ CJK อย่างถูกต้องใน hook / sandbox
- **ความเข้ากันได้ของวิธีการป้อนข้อมูลภาษาจีน (IME)**: ความล่าช้าในการป้อนและการสั่นของเคอร์เซอร์ใน TUI ขณะหน้าต่างตัวเลือก IME ทำงาน

บันทึกการแก้ไขเฉพาะและการทดสอบถดถอยดูได้ที่ [`docs/localization/zh-hans-fixes.md`](./docs/localization/zh-hans-fixes.md)

> 💡 หากคุณพบปัญหาด้านภาษาจีนอื่น ๆ ระหว่างการใช้งาน กรุณาส่งขั้นตอนการทำซ้ำใน [พื้นที่ issue](./issues) ฉันจะแก้ไขข้อบกพร่องอย่างต่อเนื่อง

## คุณสมบัติที่สืบทอดจากต้นน้ำ

คุณสมบัติต่อไปนี้มาจาก OpenCode ต้นน้ำ (ใบอนุญาต MIT) ทั้งหมด สาขานี้ไม่ได้ทำการแก้ไขด้านฟังก์ชัน:

### แอปพลิเคชันเดสก์ท็อป (BETA)

OpenCode ยังมีแอปพลิเคชันเดสก์ท็อปด้วย สามารถดาวน์โหลดได้จาก [หน้าเผยแพร่](https://github.com/anomalyco/opencode/releases) หรือ [opencode.ai/download](https://opencode.ai/download)

| แพลตฟอร์ม             | ไฟล์ดาวน์โหลด                            |
| --------------------- | ---------------------------------------- |
| macOS (Apple Silicon) | `opencode-desktop-darwin-aarch64.dmg`    |
| macOS (Intel)         | `opencode-desktop-darwin-x64.dmg`        |
| Windows               | `opencode-desktop-windows-x64.exe`       |
| Linux                 | `.deb`, `.rpm` หรือ AppImage             |

```bash
# macOS (Homebrew Cask)
brew install --cask opencode-desktop
# Windows (Scoop)
scoop bucket add extras; scoop install extras/opencode-desktop
```

### Agents

OpenCode มี Agent ในตัวสองตัว ใช้ปุ่ม `Tab` เพื่อสลับอย่างรวดเร็ว:

- **build** - โหมดเริ่มต้น มีสิทธิ์เต็มที่ เหมาะสำหรับการพัฒนา
- **plan** - โหมดอ่านอย่างเดียว เหมาะสำหรับการวิเคราะห์และสำรวจโค้ด
  - ปฏิเสธการแก้ไขไฟล์โดยค่าเริ่มต้น
  - จะถามก่อนรันคำสั่ง bash
  - เหมาะสำหรับการสำรวจฐานโค้ดที่ไม่รู้จักหรือวางแผนการเปลี่ยนแปลง

นอกจากนี้ยังมี sub-Agent **general** สำหรับการค้นหาที่ซับซ้อนและงานหลายขั้นตอน ใช้ภายใน และสามารถเรียกใช้ได้โดยพิมพ์ `@general` ในข้อความ

เรียนรู้เพิ่มเติมเกี่ยวกับ [Agents](https://opencode.ai/docs/agents)

### ClaudeCode Hooks API การนำไปใช้แบบซุปเปอร์เซต

สาขานี้รักษาระบบ Hooks API และเหตุการณ์เรียกใช้งานรันไทม์ 22 เหตุการณ์จากต้นน้ำอย่างครบถ้วน Hook ลงทะเบียนในช่อง `hooks` ของไฟล์การตั้งค่าตามชื่อเหตุการณ์ รองรับประเภทการดำเนินงาน 5 ประเภท: `command`, `mcp`, `http`, `prompt`, `agent` และสื่อสารผ่านซอง JSON ทาง stdin/stdout ดูโปรโตคอลฉบับเต็มได้ที่ [`packages/opencode/src/session/prompt/hooks-reference.md`](./packages/opencode/src/session/prompt/hooks-reference.md)

ดูรายการเหตุการณ์รายละเอียดและตารางประเภทการดำเนินงานที่ [ส่วนคุณสมบัติต้นน้ำ](./docs/readmes/upstream-features.md)

2. **ข้อกำหนดการแพร่กระจายของ AGPL** — โครงการใดๆ ที่ใช้ แก้ไข หรือพัฒนาต่อยอดจากโมดูล AGPL-3.0 (เช่น DAG workflow engine) **ต้องเปิดเผยซอร์สโค้ดทั้งหมดภายใต้ AGPL-3.0** และต้องให้ผู้ใช้ปลายทางเข้าถึงได้
3. **SaaS ต้องเปิดซอร์ส** — หากคุณปรับใช้โครงการนี้หรืองานดัดแปลงเป็นบริการเครือข่าย (SaaS / แพลตฟอร์มคลาวด์) คุณ**ต้องให้ลิงก์ดาวน์โหลดซอร์สโค้ดทั้งหมดแก่ผู้ใช้ทุกคนของบริการนั้น** (นี่เป็นข้อกำหนดสำคัญที่แยก AGPL ออกจาก GPL, มาตรา 13)
4. **การสงวนชื่อผู้แต่ง** — ต้องสงวนประกาศผู้เขียนต้นฉบับ การระบุลิขสิทธิ์ และข้อมูลการระบุแหล่งที่มาในไฟล์ NOTICE

### ความสัมพันธ์กับทีม opencode ดั้งเดิม

- ✅ โครงการนี้ **สร้างบน** โค้ดต้นน้ำ [opencode](https://github.com/sst/opencode)
- ❌ โครงการนี้ **ไม่มีความสัมพันธ์ใดๆ** อย่างเป็นทางการหรือได้รับอนุญาตจากทีม opencode อย่างเป็นทางการ (sst / anomalyco)
- ❌ โครงการนี้ไม่ใช่รุ่นเผยแพร่อย่างเป็นทางการของ opencode และไม่มีข้อผูกมัดในการสนับสนุนต้นน้ำอย่างเป็นทางการ
- ❌ **ทีม OpenCode อย่างเป็นทางการไม่ให้การสนับสนุนทางเทคนิค การรับประกัน หรือการรับรองใดๆ สำหรับสาขานี้** (ตามข้อกำหนดการระบุแหล่งที่มาใน README ต้นน้ำ)
- ✅ DAG workflow engine, การดีบักคุณลักษณะภาษาจีน และการปรับปรุงอื่นๆ ของโครงการนี้ได้รับการดูแลโดยอิสระจากผู้เขียน
- ✅ การระบุแหล่งที่มาของโค้ด MIT ต้นน้ำถูกสงวนไว้อย่างสมบูรณ์ ไม่มีการดัดแปลงประกาศผู้เขียนและลิขสิทธิ์

## คำถามที่พบบ่อย (FAQ)

### นี่แตกต่างจาก Claude Code อย่างไร?

ในแง่ฟังก์ชันมีความคล้ายคลึงกันมาก ความแตกต่างหลัก:

- โอเพนซอร์ส 100%
- ไม่ผูกมัดกับผู้ให้บริการเฉพาะ แนะนำให้ใช้โมเดลจาก [OpenCode Zen](https://opencode.ai/zen) แต่สามารถใช้ร่วมกับ Claude, OpenAI, Google หรือแม้กระทั่งโมเดลโลคัล
- มี LSP ในตัว
- เน้นส่วนติดต่อเทอร์มินัล (TUI)
- สถาปัตยกรรมไคลเอนต์/เซิร์ฟเวอร์ รันบนเครื่องท้องถิ่นและควบคุมระยะไกลด้วยอุปกรณ์มือถือ
- **🪝 Hooks API แบบ superset**: ต่อยอดจากเหตุการณ์ทริกเกอร์ 22 แบบ × ประเภทการรัน 5 แบบของ Claude Code โดย fork นี้**เข้ากันได้กับ Claude Code Hooks protocol อย่างสมบูรณ์** และเพิ่มการรวม event bus ของ DAG workflow (`workflow.*` / `node.*`), การ subscribe ใน TUI และการ forward ผ่าน HTTP API ดูสเปกเต็มได้ที่ [`hooks-reference.md`](./packages/opencode/src/session/prompt/hooks-reference.md)
- **🎯 ระบบคำสั่ง Goal**: เครื่องมือ `todowrite` + การติดตามเป้าหมายแบบมีโครงสร้าง ช่วยเก็บคิวงานของ agent ระหว่างงานหลายขั้นตอนระยะยาว และลดการสูญเสียสถานะงานเมื่อ context window เปลี่ยน
- **🪝 TODO PreHook**: รองรับการ inject รายการ TODO เข้า context ผ่าน hook `PreToolUse`; กลไก goal reentry ที่ขับเคลื่อนด้วย hooks ทำให้ agent เห็นความคืบหน้าปัจจุบันเสมอ
- **🛡️ Sandbox Coding Workspace**: sandbox แต่ละตัวมีไดเรกทอรีชั่วคราวแยก, LSP diagnostics และ toolchain หลายภาษา (Python/Node/TS/Go/Rust/C/C++); agent สามารถทดลอง คอมไพล์ และดีบักโค้ดแบบแยกส่วน ก่อนรวมกลับเข้าไฟล์โปรเจกต์ผ่าน edit/write

### นี่แตกต่างจากเวอร์ชันทางการของ opencode อย่างไร?

- **🪝 Hooks API superset + Goal instructions + TODO PreHook + Sandbox Workspace**: รักษาความสามารถ Hooks จาก upstream ทั้งหมด และเพิ่มการรวม event ของ DAG, การติดตามงานแบบมีโครงสร้าง, goal reentry ผ่าน hooks และ coding sandbox หลายภาษาแบบแยกส่วน
- **🧩 โหมด DAG WorkFlow (WIP · เสร็จประมาณ 90%)**: เอนจิน [Harness-DAG-Workflow](./docs/harness-dag.md) ที่พัฒนาเอง ช่วยให้ LLM agent จัดการงานขนานหลายโหนดใน session เดียวได้ ความสามารถหลักเสร็จแล้ว (scheduling / lifecycle / pause-resume-cancel-replan-step / sub-DAG / conditional branching / data flow / crash recovery / probes), แผง TUI เชื่อมต่อแล้ว และกำลังเก็บรายละเอียดส่วนที่เหลือ
- **🔧 การแก้ไขความเข้ากันได้ของภาษาจีน**: DEBUG ต่อเนื่องสำหรับ CJK tokenization, เครื่องหมาย fullwidth, path ภาษาจีน และ IME edge cases จาก upstream
- ดูแลอย่างอิสระในระยะยาว ไม่ขึ้นกับจังหวะของต้นน้ำ
