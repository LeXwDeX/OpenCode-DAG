Use this tool when you need to ask the user questions during execution. This allows you to:
1. Gather user preferences or requirements
2. Clarify ambiguous instructions
3. Get decisions on implementation choices as you work
4. Offer choices to the user about what direction to take.

REQUIRED fields per question item (DO NOT OMIT):
- question (string)  The full question sentence the user will read.
- header   (string)  A very short label (max 30 chars). It is a badge/title, NOT a substitute for "question".
- options  (array)   Each option must have { label: string, description: string }.

Optional fields:
- multiple (boolean) Allow selecting multiple choices. Default false.
- custom   (boolean) Default true. When true, a "Type your own answer" option is auto-added; do NOT include "Other" or catch-all options yourself.

Common mistakes to avoid:
- Putting the question text into "header" and leaving "question" empty/undefined. Both are required and serve different purposes.
- Forgetting "description" inside an option object.
- Returning a single object instead of an array for "questions".

Recommendation styling:
- If you recommend a specific option, place it first and append "(Recommended)" to its label.

Answers:
- Returned as arrays of selected labels. Set "multiple": true to allow more than one selection.

Correct call example:
{
  "questions": [
    {
      "question": "Which deployment target should we use?",
      "header": "Deploy target",
      "options": [
        { "label": "Vercel (Recommended)", "description": "Edge runtime, zero-config" },
        { "label": "Cloudflare Pages", "description": "Workers + KV integration" }
      ],
      "multiple": false
    }
  ]
}
