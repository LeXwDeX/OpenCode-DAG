Use this tool when you need to ask the user questions during execution. This allows you to:
1. Gather user preferences or requirements
2. Clarify ambiguous instructions
3. Get decisions on implementation choices as you work
4. Offer choices to the user about what direction to take.

Usage notes:
- When `custom` is enabled (default), a "Type your own answer" option is added automatically; don't include "Other" or catch-all options
- Answers are returned as arrays of labels; set `multiple: true` to allow selecting more than one
- If you recommend a specific option, make that the first option in the list and add "(Recommended)" at the end of the label

## Parameters

- `question` (REQUIRED): The question text to display. NOT the same as `header`.
- `header` (REQUIRED): A short label (max 30 chars) shown above the question.
- `options` (optional): Array of { label: string, description: string } choices.
- `multiple` (optional, boolean): Allow selecting more than one choice.

## Common Mistakes

- Using `header` as the question text — header is just a label; the actual question goes in `question`.
- Forgetting `options` when you want to offer choices.

## Correct Call Example

{
  "question": "Which database should we use?",
  "header": "Database Choice",
  "options": [
    { "label": "PostgreSQL", "description": "Most robust, great for complex queries" },
    { "label": "SQLite", "description": "Lightweight, zero-config, great for single-user" }
  ]}
