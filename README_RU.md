# OpenCode (Расширенное издание)

> **⚠️ Заявление**: Этот проект является оптимизированным форком [opencode](https://github.com/sst/opencode), поддерживаемым независимым разработчиком на основе оригинала. Этот проект **не имеет никакого отношения к официальной команде OpenCode** и никак с ней не связан. Оригинальный проект был выпущен под лицензией MIT командой opencode. Этот форк сохраняет оригинальную лицензию MIT и добавляет несколько самостоятельно разработанных модулей (подробности в [NOTICE](./NOTICE)).

## Введение

Этот проект представляет собой **переработанную и улучшенную версию** официальной версии opencode со следующими целями:

- 🔧 **Исправление проблем с китайским языком**: Отладка ряда проблем совместимости с китайской токенизацией, обработкой символов CJK, полноширинной пунктуацией, китайскими путями к файлам и китайским методом ввода (IME) в upstream-версии (см. [список исправлений китайских языковых функций](./docs/localization/zh-hans-fixes.md))
- 🧩 **Предоставление продукционного движка рабочих процессов DAG**: Собственная разработка [Harness-DAG-Workflow](./docs/harness-dag.md), позволяющая LLM-агенту оркестрировать и выполнять многоузловые параллельные задачи в рамках одной сессии
- 🎯 **Сохранение совместимости с upstream**: Весь upstream-код под лицензией MIT остаётся без изменений — без нарушения существующих сборок и без загрязнения upstream API

## Установка

```bash
# Прямая установка (YOLO)
curl -fsSL https://opencode.ai/install | bash

# Менеджеры пакетов
npm i -g opencode-ai@latest        # также можно использовать bun/pnpm/yarn
scoop install opencode             # Windows
choco install opencode             # Windows
brew install anomalyco/tap/opencode # macOS и Linux (рекомендуется, всегда актуальная версия)
brew install opencode              # macOS и Linux (официальная формула brew, обновляется реже)
sudo pacman -S opencode            # Arch Linux (Stable)
paru -S opencode-bin               # Arch Linux (Latest from AUR)
mise use -g opencode               # Любая система
nix run nixpkgs#opencode           # Или используйте github:anomalyco/opencode для последней dev-ветки
```

> [!TIP]
> Перед установкой удалите старые версии (до 0.1.x).

### 🧩 Система оркестрации задач DAG HARNESS (собственный модуль · AGPL-3.0)

Ранее назвался Harness-DAG-Workflow. Это промышленный **движок рабочих процессов на основе ориентированного ациклического графа (DAG)**, позволяющий LLM-агенту оркестрировать сложные параллельные задачи в рамках одной сессии. Ключевые возможности:

- **Автоматическое планирование**: на основе зависимостей узлов автоматически порождает подагентов и выполняет их параллельно
- **Динамическое перепланирование**: позволяет перестраивать рабочий процесс в реальном времени (добавление/удаление узлов, изменение максимальной степени параллелизма)
- **Строгое соблюдение правил**: конечный автомат нельзя обойти, конечное состояние необратимо, события транслируются обязательно, приоритет отдаётся персистентности
- **Интеграция Slash-команд**: `/dag-ctl` для управления выполнением, `/dag-worker` для конфигурирования рабочего процесса
- **Аудит с персистентностью**: схема из 6 таблиц SQLite, все изменения состояний отслеживаются

Полное описание архитектуры см. в [документации Harness-DAG-Workflow](./docs/harness-dag.md), руководство для разработчиков — в [AGENTS.md](./packages/opencode/src/dag/AGENTS.md).

> **Лицензия**: Этот модуль ([`packages/opencode/src/dag/`](./packages/opencode/src/dag/), [`packages/opencode/src/tool/dag-worker.ts`](./packages/opencode/src/tool/dag-worker.ts), [`packages/opencode/src/tool/node_complete.ts`](./packages/opencode/src/tool/node_complete.ts) и связанные шаблоны и документация) распространяется под лицензией **GNU AGPL v3** — использование этого модуля требует открытия исходного кода всех модификаций. Подробнее см. в [NOTICE](./NOTICE) и [packages/opencode/src/dag/LICENSE](./packages/opencode/src/dag/LICENSE)

### 🪝 Реализация надмножества HOOKS API

Данная ветка полностью сохраняет и расширяет систему Hooks API от upstream:

- **22 события времени выполнения**: `PreToolUse` / `PostToolUse` / `FileChanged` / `UserPromptSubmit` / `Stop` / `SessionStart` / `SessionEnd` / `PermissionRequest` / `PermissionDenied` / `SubagentStart` / `SubagentStop` / `TaskCreated` / `TaskCompleted` / `TeammateIdle` / `PreCompact` / `PostCompact` / `WorktreeCreate` / `WorktreeRemove` / `ConfigChange` / `InstructionsLoaded` / `StopFailure` / `PostToolUseFailure`
- **5 типов исполнения**: `command` (shell) / `mcp` (MCP-инструмент) / `http` (REST) / `prompt` (одношаговый LLM) / `agent` (многошаговый LLM)
- **Протокол обмена JSON-конвертами через stdin/stdout**: полная документация в [`hooks-reference.md`](./packages/opencode/src/session/prompt/hooks-reference.md)
- **Улучшения в данной ветке**: интеграция с шиной событий рабочих процессов DAG (события `workflow.*` / `node.*`) + подписка в TUI + пересылка через HTTP API

### 🛡️ Легковесное изолированное пространство CODING

Данная ветка предоставляет изолированную среду выполнения с двумя уровнями изоляции, позволяющую агенту/пользователю безопасно выполнять код в песочнице, не загрязняя реальный репозиторий:

| Уровень изоляции | Механизм | Назначение |
|------------------|----------|------------|
| **Sandbox** (легковесный) | Временная директория + диагностика LSP + инструментарий для нескольких языков (Python/Node/TS/Go/Rust/C/C++) | Прогон кода для одиночных файлов / небольших экспериментов |
| **Worktree** (тяжёлый) | `git worktree` с независимой веткой и изолированным представлением файловой системы | Параллельное редактирование несколькими агентами, масштабный рефакторинг |

- 📦 **Инструмент Sandbox**: `packages/opencode/src/tool/sandbox.ts`, каждая песочница имеет независимый кэш зависимостей (venv / node_modules), поддерживает одноразовый режим `ephemeral` и фоновые асинхронные длительные задачи `background`
- 🌳 **Менеджер DAG Worktree**: в рабочем процессе DAG каждый параллельный узел может быть автоматически размещён в независимой ветке worktree, а по завершении узла изменения сливаются в основную ветку через `git merge`

### 🔧 Отладка функций для китайского языка (исправленные проблемы upstream)

Была проведена отладка и оптимизация ряда проблем совместимости и удобства использования, обнаруженных в upstream-версии при работе с китайским языком, охватывающая:

- **Сегментацию китайского текста и подсчёт токенов**: некорректная обработка символов CJK в некоторых токенизаторах
- **Совместимость с полноширинными знаками пунктуации**: устойчивость к ошибкам при разборе конфигураций, содержащих полноширинные двоеточия, кавычки, скобки
- **Обработка путей с китайскими символами**: корректная передача путей файлов, содержащих пробелы и символы CJK, в перехватчиках / песочнице
- **Совместимость с методами ввода китайских иероглифов (IME)**: задержка ввода и дрожание курсора в TUI при отображении окна кандидатов IME

Конкретные записи об исправлениях и регрессионные тесты см. в [`docs/localization/zh-hans-fixes.md`](./docs/localization/zh-hans-fixes.md).

> 💡 Если вы обнаружите другие проблемы, связанные с китайским языком, пожалуйста, сообщите о них в [разделе issues](./issues) с шагами для воспроизведения, я продолжу отладку.

### 🔬 Прочие мелкие исправления (интегрированные исправления upstream)

Данная ветка полностью сохраняет ряд исправлений мелких проблем юзабилити из upstream, прошедших регрессионное тестирование:

| Проблема | Коммит исправления upstream | Область влияния |
|----------|-----------------------------|-----------------|
| 📋 **Повреждение содержимого при копировании/вставке** — вставляемый пользователем промпт некорректно обрезается или теряет символы в TUI | `564cde393e` [#28156](https://github.com/anomalyco/opencode/pull/28156) | Ввод в TUI |
| 📐 **Макет не обновляется после вставки** — после вставки длинного текста высота поля промпта не подстраивается автоматически, создавая визуальный эффект обрезки | `1124315267` [#28164](https://github.com/anomalyco/opencode/pull/28164) | Ввод в TUI |
| 📎 **Отсутствие fallback при неудачной записи в буфер обмена** — когда API `navigator.clipboard` терпит неудачу (например, в HTTP-окружении), операция копирования завершается ошибкой | `fe143df151` [#27993](https://github.com/anomalyco/opencode/pull/27993) | Кросc-браузерная совместимость |
| 🎨 **Недостаточная контрастность цвета бейджа вставки** — текст сводки операции вставки трудночитаем в некоторых темах | `e94aecaa08` [#27969](https://github.com/anomalyco/opencode/pull/27969) | Визуальный опыт в TUI |
| 📏 **Некорректная оценка ширины символов CJK / восточноазиатских языков** — отображаемая ширина эмодзи, полноширинных символов, иероглифов и других восточноазиатских символов не соответствует фактическому занимаемому месту, что приводит к смещению курсора | Включено в систему исправлений сегментации CJK | Выравнивание символов в TUI |
| ⌨️ **Дрожание окна кандидатов IME** — при активации методов ввода китайского/японского языка наблюдается дрожание курсора и задержка вставки символов | Локальный патч-обход (workaround) | Ввод в TUI |

> Данная ветка не изобретает велосипед: исправления, уже сделанные в upstream, будут подтягиваться при синхронизации с веткой `stable`; основное внимание в этой ветке уделяется отладке проблем, связанных с китайским языком и рабочими процессами DAG, ещё не обработанных в upstream.

### Агенты

В OpenCode встроены два агента. Используйте клавишу `Tab` для переключения между ними:

- **build** — Режим по умолчанию с полными правами, подходит для разработки
- **plan** — Режим только для чтения, подходит для анализа и исследования кода
  - По умолчанию отклоняет изменения файлов
  - Спрашивает перед выполнением команд bash
  - Удобен для исследования неизвестных кодовых баз или планирования изменений

Также включает подагент **general** для сложных поисков и многошаговых задач, используемый внутренне, который также можно вызвать, введя `@general` в сообщении.

Узнайте больше об [агентах](https://opencode.ai/docs/agents).

### Реализация надмножества ClaudeCode Hooks API

Этот форк полностью сохраняет upstream-систему Hooks API и 22 события-триггера времени выполнения. Хуки регистрируются в поле `hooks` конфигурационного файла по имени события и поддерживают пять типов выполнения: `command`, `mcp`, `http`, `prompt` и `agent`. Коммуникация осуществляется через JSON-конверт по stdin/stdout. Полная спецификация протокола в [`packages/opencode/src/session/prompt/hooks-reference.md`](./packages/opencode/src/session/prompt/hooks-reference.md).

Подробный список событий и таблица типов выполнения в [разделе сохранения upstream-функций](./docs/readmes/upstream-features.md).

## Часто задаваемые вопросы (FAQ)

### В чём отличие от Claude Code?

Функционально довольно похоже, с ключевыми отличиями:

- 100% открытый исходный код
- Не привязан к конкретному провайдеру. Рекомендуются модели [OpenCode Zen](https://opencode.ai/zen), но также можно использовать с Claude, OpenAI, Google или локальными моделями
- Встроенная поддержка LSP
- Фокус на терминальном интерфейсе (TUI)
- Архитектура клиент/сервер. Может работать локально и одновременно управляться удалённо с мобильного устройства

### В чём отличие от официальной версии opencode?

- Новый движок рабочих процессов Harness-DAG-Workflow (AGPL-3.0)
- Постоянная отладка проблем совместимости в китайских сценариях использования
- Долгосрочное независимое обслуживание, не зависящее от темпа upstream
