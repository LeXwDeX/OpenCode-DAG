或者像OPUS4-8这种强大的模型还会出现这种奇怪的CALL TOOLS：
```
开始 WP1.1：TUI pause/resume-bar 接入。先 explore TUI DAG 现状，定位 pause-resume-bar 应该接入的位置和现有数据流。
call explore：
call concurrently 探查 TUI DAG 结构和 pause-resume-bar 现状。
call the explore subagent now.
call task explore.
I'll launch the explore agent to map the TUI DAG structure.
call task.
(emitting tool call)
Let me dispatch explore.
call now.
call:
call task explore subagent.
call.
I'll call it.
call task.
OK — executing.
call the task tool.
calling now.
call task tool with explore subagent.
...如此循环
```
经过调查是：这轮对话处在 tool-use loop 中，还开启了 thinking。Anthropic 文档说明，thinking + tool use 只能配 tool_choice: auto/none，不能强制某个工具；并且不能在 tool-use loop 中途切换 thinking 模式，否则可能静默禁用 thinking。日志里响应确实没有 thinking block，符合“thinking 被禁用/降级”的迹象。加上上下文污染，最终变成普通文本循环。

需要客户侧调整：
不要在正在进行的 tool-use loop 中途切换 thinking 模式。按 Anthropic 规范，应在完成整个 assistant turn 之后（等 message_stop 到来，stop_reason !=tool_use）再切换 thinking 设置；如果需要在 thinking 开启下使用：要么全程开启 thinking，要么在 conversation 的第一轮 user 消息就带上 thinking 配置，不要在工具循环中间插入。
```
