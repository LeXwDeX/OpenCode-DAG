# Qwen3.7-Max 异常停止问题 - 复现计划

## 目标
模拟和复现 qwen3.7-max 模型在多轮对话中异常停止的问题，收集完整的证据链。

## 复现环境

### 1. 基础环境
- OpenCode 客户端（stable 分支）
- DCP 插件（fix/preserve-state-and-continuation-nudge 分支）
- 模型：qwen3.7-max（通过 bailian-token-plan）

### 2. 配置要求
- 启用 DCP 插件
- 使用英文提示词（触发问题）
- 启用详细日志记录

## 复现步骤

### Phase 1: 环境准备
1. 确认 OpenCode 和 DCP 插件已正确安装
2. 配置 qwen3.7-max 模型
3. 启用 DCP 插件（使用英文提示词版本）
4. 配置日志级别为 DEBUG

### Phase 2: 触发问题
1. 启动 OpenCode
2. 发送多步骤任务（需要工具调用）
3. 观察模型行为
4. 记录异常停止的时间点

### Phase 3: 证据收集
1. 收集 OpenCode 日志文件
2. 提取关键日志行（finishReason、hasToolCalls、output tokens）
3. 记录模型的实际输出内容
4. 截图 TUI 界面（如果可能）

### Phase 4: 对比测试
1. 使用中文提示词版本测试（应该减少问题）
2. 使用其他模型测试（如 Claude、GPT-4）
3. 对比不同配置下的行为差异

## 预期结果

### 问题复现时
- 模型在 step 2-4 之间停止
- `finishReason=stop`
- `hasToolCalls=false`
- `output=2`（只生成 2 个 token）
- 任务未完成

### 修复后
- 模型正常执行任务
- 调用必要的工具
- 任务完成

## 成功标准
1. 成功复现问题（至少 3 次）
2. 收集到完整的日志证据
3. 确认根本原因
4. 验证修复方案的有效性

- 证据收集：15 分钟
